# VALOR Desktop Client – Setup

## Prerequisites

| Tool | Minimum version | Notes |
|------|----------------|-------|
| Node.js | 20 LTS | 22 LTS recommended |
| npm | 10 | bundled with Node 20+ |
| Git | any | for cloning / version control |
| Windows | 10 x64 | primary dev target |
| Linux | Ubuntu 22+ / Debian 12+ | secondary target |

> **No Rust, Python, or native build tools are required.** electron-builder downloads pre-built NSIS on first use.

---

## Clone / first install

```bash
# If you cloned the mono-repo:
cd c:\VALOR-Client

# Install all dependencies (Electron, React, TypeScript, Tailwind, etc.)
npm install
```

`npm install` will also download the Electron binary for your platform (cached in `~/.electron`).

---

## Environment — is there anything to configure?

**Nothing for development.** The API base URL (`https://apiv.dawn-star.co.uk`) and the CORS origin are hard-coded in `src/main/index.ts` and `src/renderer/src/services/api.ts`. No `.env` file is needed.

If you need to point the client at a **different backend** (e.g. a local dev server on port 3001):

1. In `src/renderer/src/services/api.ts`, change `API_BASE`:
   ```typescript
   export const API_BASE = 'http://localhost:3001'
   ```
2. In `src/main/index.ts`, update both constants:
   ```typescript
   const API_ORIGIN = 'http://localhost:2903'
   const API_BASE   = 'http://localhost:3001'
   ```
3. In `src/renderer/index.html`, relax the CSP `connect-src` to include the new origin.

## Running in development

```bash
npm run dev
```

This command (via `electron-vite dev`) does three things in parallel:
1. Compiles `src/main/index.ts` and `src/preload/index.ts` into `out/` using esbuild.
2. Starts a Vite dev server on `http://localhost:5173` serving the React renderer.
3. Launches Electron, which loads `http://localhost:5173` in the `BrowserWindow`.

Hot-module replacement (HMR) is active for the renderer — React changes appear instantly without restarting Electron. Main process changes require a full restart (`Ctrl+C` → `npm run dev`).

**DevTools** open automatically in detach mode. Press `F12` inside the window to focus them.

## Adding an app icon

The installer config references `build/icon.ico` (Windows) and `build/icon.png` (Linux). These are **not included** in the repo — add your own:

| File | Size | Format | Used for |
|------|------|--------|---------|
| `build/icon.ico` | 256×256 (multi-resolution) | `.ico` | Window title bar, taskbar, installer |
| `build/icon.png` | 512×512 | `.png` | Linux AppImage / .deb |

Until you add icons, electron-builder uses Electron's default icon (the blue hexagon). The app will still build and run correctly.

Free tools for generating `.ico` from a PNG: [favicon.io](https://favicon.io), [icoconvert.com](https://icoconvert.com).

## Common issues

### `npm install` fails on Windows

Make sure you have no spaces in the path to `c:\VALOR-Client`. If you do, move the folder.

### Electron window doesn't open

Check the terminal for compile errors. The main process must compile cleanly before Electron launches. Run `npm run build` to see TypeScript errors directly.

### API calls return 403

You're hitting a CORS issue. In dev mode, check that `src/main/index.ts` has the correct `API_BASE` for `session.webRequest.onBeforeSendHeaders`. The filter URL must match the actual API host exactly.

### `electron-store` type errors

`electron-store@8` is CommonJS. If you see ESM import errors, ensure `tsconfig.node.json` has `"esModuleInterop": true` and `"moduleResolution": "bundler"`.

### White/blank renderer screen

Open DevTools (auto-opens in dev mode) and check the console. Common causes: syntax error in `main.tsx`, failed import, or unhandled React error boundary.

### Watchlist page goes blank / shows nothing

This is caused by the API service receiving a wrapped response `{ success, items }` and calling `.map()` on the wrapper object instead of the array inside it. The fix is in `src/renderer/src/services/api.ts` — `getWatchlist()` must unwrap `res.items`:
```typescript
const res = await request<{ success: boolean; items: UnifiedMedia[] }>('/watchlist')
return res.items ?? []
```
If you see a blank page on any route, check the browser console for `TypeError: data.map is not a function` — that's always this same root cause.

### Profile photos not showing

Two combined issues:
1. **Response shape**: `GET /me` returns `{ success: true, user: { ... } }` — not a bare `User` object. Code that does `const user = await getMe()` and then reads `user.avatarUrl` directly was actually reading `undefined` because `user` was the whole wrapper.
2. **Avatar URL rewriting**: The backend stores avatar paths as `/api/avatars/filename.jpg` (the Next.js frontend's proxy path). The desktop client calls the backend directly, so the URL must be rewritten to `https://apiv.dawn-star.co.uk/avatars/filename.jpg`. The `resolveAvatarUrl()` helper in `api.ts` handles this.

Both are fixed at the `api.ts` service layer — `normalizeUser()` unwraps and rewrites all user objects coming from the backend.

### `electron-vite` not recognized on first run

Run `npm install` — the binary lives in `node_modules/.bin/electron-vite`. The build pipeline (`scripts/build-win.mjs`) detects a missing `node_modules` folder and runs `npm install` automatically, but the dev script does not.

### Build fails with symlink / elevated privileges error (winCodeSign)

See [BUILD.md → Known build issues → winCodeSign macOS symlink error](./BUILD.md#2-wincodeSign-macos-symlink-error-elevated-privileges). The `scripts/build-win.mjs` pipeline resolves this automatically via `ensureWinCodeSign()`.

### Auto-update shows 404 for `latest.yml`

The backend `/updates/` static route is not configured. See [BUILD.md → Known build issues → Auto-update returns HTTP 404](./BUILD.md#4-auto-update-returns-http-404-cannot-find-channel-latestyml).
